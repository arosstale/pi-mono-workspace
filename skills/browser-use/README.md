# browser-use Skill

Natural language browser automation - no Playwright scripts needed.

---

## Quick Start

Instead of writing code, just describe what you want:

> "Go to Reddit r/openclaw and extract the top post title and upvotes"

The AI figures out:
- Which URL to navigate to
- Which elements to click
- How to extract the data
- How to handle errors

---

## Why It Matters

| Traditional | browser-use |
|-------------|--------------|
| Hours of coding | Minutes |
| Fragile (breaks on updates) | Adapts to changes |
| Debug DOM selectors | Natural language |
| Requires coding skill | Anyone can use |

## Examples

### Job Search
> "Go to LinkedIn Jobs, search for 'AI Engineer' in San Francisco, and extract company names, job titles, and application links from the first 10 results"

### E-commerce Scraping
> "Go to Amazon, search for 'MacBook Pro M3', filter by 4+ stars, and extract the top 5 product names, prices, and ratings"

### Social Media
> "Go to Twitter/X and extract the latest 10 tweets from @OpenAI including text, likes, and retweets"

### Lead Generation
> "Go to this trade show directory, extract all company names, websites, and booth numbers"

### Price Monitoring
> "Check competitor prices on these 3 e-commerce sites and create a comparison table"

## Use Cases

- **Lead Generation** - Scrape trade show directories, business listings

- **Job Hunting** - Aggregate job postings from multiple sites
- **Content Monitoring** - Track news, social media, blog posts
- **Testing** - Walk through user flows, verify functionality
- **Data Entry** - Automate form filling across multiple sites
